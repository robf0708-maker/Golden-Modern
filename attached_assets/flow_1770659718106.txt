📋 CENÁRIOS DE RISCO POR CATEGORIA
CATEGORIA 1: CLASSIFICAÇÃO DE INTENT (IA)
RISCO 1.1: IA classificar errado "quantidade"
Cenário:
Cliente: "Quero 3 cortes" (pra ele, esposa e filho)
IA classifica: select_service (1 corte)
Sistema agenda: 1 corte
Por quê pode acontecer?

Intent atual não tem campo quantity
Lógica de companion só detecta "meu filho", não "3 pessoas"

Como prevenir:
typescript// Adicionar ao CLASSIFY_INTENT_TOOL:
{
  name: "book_multiple_people",
  description: "Cliente quer agendar para N pessoas",
  parameters: {
    quantity: "number",  // 2, 3, 4...
    relationship: "string[]"  // ["filho", "esposa"]
  }
}

RISCO 1.2: IA não detectar remarcação
Cenário:
Cliente: "Mudar meu corte de amanhã pra sexta"
IA classifica: unclear
Sistema: "Pode explicar melhor?"
Por quê pode acontecer?

Não existe intent reschedule_appointment
IA tenta encaixar em cancel_appointment + select_service

Como prevenir:
typescript{
  name: "reschedule_appointment",
  parameters: {
    appointment_index: "number",
    new_date: "YYYY-MM-DD",
    new_time: "HH:MM"
  }
}

RISCO 1.3: Cliente pede "mais cedo" mas não especifica quanto
Cenário:
Bot: "Próximo horário: 16h"
Cliente: "Tem mais cedo?"
IA classifica: reject_time ✅
Sistema busca: desde 08:00 de novo ❌
Por quê pode acontecer?

reject_time não tem parâmetro "direção" (mais cedo vs mais tarde)
Sistema não sabe se cliente quer < 16h ou > 16h

Como prevenir:
typescript// Adicionar campo ao reject_time:
{
  name: "reject_time",
  parameters: {
    reason: "earlier" | "later" | "other",
    preferred_time: "HH:MM" // se mencionar
  }
}

  'produto', 'comprar', 'vender',

  'como funciona', 'explicar',
